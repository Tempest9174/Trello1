# DB設計・画面遷移 学習メモ

設計中に出た疑問と回答をまとめた参照用ドキュメントです。

## Q1. ホーム・ボード・カード・ラベルって何？

### ホーム画面
アプリを開いたときに最初に表示される画面。作成したボードの一覧が並んでいます。

### ボード（Board）
作業スペース全体のこと。「仕事用」「個人用」など用途ごとに複数作れます。
ボードの中に複数のリスト（列）が並びます。

```
ボード「仕事用」
  ├─ リスト「やること」
  ├─ リスト「進行中」
  └─ リスト「完了」
```

### カード（Card）
1件のタスクを表す四角いメモ。リストの中に並びます。
タイトル・説明・期限日・優先度・ラベルを設定できます。

### ラベル（Label）
カードに貼る色付きタグ。「バグ」「重要」「確認待ち」などの分類に使います。
ボードごとに作成・管理され、1枚のカードに複数貼れます。

### 全体の親子構造

```
ホーム画面
  └─ ボード（複数）
       └─ リスト（複数の列）
            └─ カード（複数のタスク）
                  └─ ラベル（複数の色タグ）※ 多対多
```

## Q2. なぜテーブルが5個必要か？（先生の1テーブルとの違い）

### 先生のアプリ（1テーブル）

シンプルなTodoリストのイメージ。

```
tasksテーブル
  id / タイトル / 説明 / ステータス / 期限日
```

ボードもリストも概念がなく、ただタスクを並べるだけ。

### 私たちのアプリ（5テーブル）

Trelloのように「ボード → リスト → カード」の階層構造があるため、
それぞれ別テーブルが必要になります。

| テーブルが分かれる理由 | 説明 |
|----------------------|------|
| boards / lists を分離 | 複数ボード・複数列という階層構造を持つため |
| cards を分離 | cardsがlistsに所属するため別テーブルが必要 |
| labels を分離 | ラベルはボード単位で再利用するため |
| card_labels（中間テーブル）を追加 | 1枚のカードに複数ラベルを付けられる（多対多）から |

**どちらが正解・不正解ではありません。アプリの機能に合わせた設計です。**

## Q3. リストテーブルとは何か？他のテーブルとの関係は？

リストテーブルはボード画面の「列（カラム）」を保存するテーブルです。

```
listsテーブルのカラム
┌──────────────────────────────────────────────────────┐
│ id       … このリスト自身の番号（主キー）             │
│ board_id … どのボードに属しているか（外部キー）       │
│ title    … 列の名前（例：「やること」「完了」）        │
│ position … 何番目の列か（左から1,2,3…の順番）          │
└──────────────────────────────────────────────────────┘
```

### 他テーブルとの関係

```
boardsテーブル          listsテーブル           cardsテーブル
─────────────          ──────────────           ──────────────
id ←──────────────── board_id（FK）
                        id ←──────────────────── list_id（FK）
                        title                     title
                        position                  description
                                                  priority
                                                  ...
```

- `lists.board_id = boards.id` … 「このリストはどのボードの列か」を示す
- `cards.list_id = lists.id` … 「このカードはどのリスト（列）に入っているか」を示す

### 具体的なデータ例

```
boardsテーブル
  id=1  「仕事用ボード」

listsテーブル（仕事用ボードの列が3つ）
  id=1, board_id=1, title="やること",  position=1
  id=2, board_id=1, title="進行中",    position=2
  id=3, board_id=1, title="完了",      position=3

cardsテーブル（「やること」列にタスクが2件）
  id=1, list_id=1, title="設計書を書く"
  id=2, list_id=1, title="テストを書く"
```

## Q4. card_labels（中間テーブル）が必要な理由

### 多対多とは？

- 1枚のカードに複数のラベルを付けられる
- 1つのラベルを複数のカードに付けられる

これを「多対多」と言います。

### 1テーブルで無理やりやると…

```
cardsテーブルに labels列を追加してしまうと：
  id=1, title="タスクA", labels="バグ,重要"
```

「バグというラベルが付いたタスクを全部検索したい」といった操作が難しくなります。
データの管理・検索のためにテーブルを分けることが必要です。

### card_labelsテーブルを使うと

```
cardsテーブル
  id=1  タスクA
  id=2  タスクB

labelsテーブル
  id=10  バグ（赤）
  id=11  重要（黄）

card_labelsテーブル
  card_id=1, label_id=10   ← タスクA ＋ バグ
  card_id=1, label_id=11   ← タスクA ＋ 重要
  card_id=2, label_id=10   ← タスクB ＋ バグ
```

「バグが付いたタスクを検索する」 = `label_id=10` の行を探すだけでOK。

### cardsテーブルのidとcard_labelsのcard_idの対応

```
cardsテーブル          card_labelsテーブル      labelsテーブル
─────────────          ──────────────────       ──────────────
id  ←──────────────── card_id                   id ←──────────
title                   label_id ───────────────►    name
description                                          color
...
```

- `cards.id` = `card_labels.card_id` … カードを特定する
- `labels.id` = `card_labels.label_id` … ラベルを特定する

## Q5. 画面遷移の矢印の意味（実線 vs 破線）

| 矢印の種類 | 意味 | 使われる場所 |
|----------|------|------------|
| 実線矢印 `──▶` | ページ全体が切り替わる | ホーム画面 ↔ ボード画面（2箇所のみ） |
| 破線矢印 `- - ▶` | モーダルが開く・閉じる（ページは変わらない） | ボード画面 ↔ カード詳細など |

### 実線矢印が使われるのはどこ？

```
ホーム画面  ──[ボードをクリック]──▶  ボード画面
ボード画面  ──[← 戻る]──────────▶  ホーム画面
```

この2本だけです。この遷移はURLが変わり、画面全体が別のページに切り替わります。

### 破線矢印が使われるのはどこ？

```
ボード画面  -- カードをクリック -->  カード詳細モーダル（開く）
カード詳細  -- × 閉じる / 保存 -->  ボード画面（モーダルが閉じる）
カード詳細  -- ＋ラベルを追加 -->   ラベル選択モーダル（開く）
ラベル選択  -- × 閉じる ────-->   カード詳細（戻る）
ラベル選択  -- ＋新しいラベル -->   ラベル作成モーダル（開く）
ラベル作成  -- 作成/キャンセル -->  ラベル選択（戻る）
```

モーダルはボード画面の上にポップアップが重なる形で表示されます。ページは変わりません。
